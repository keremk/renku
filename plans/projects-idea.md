I want to create a concept of a project in Renku. They will live under a projects folder under storage root. The idea behind a project is: 
1) They will contain multiple blueprints each tackling different tasks of an overall video creation project 
2) They will have a shared area where they store the generated artifacts. This is a project wide artifact storage area. 
3) These artifacts can be used as inputs to other blueprints in the project. But the blueprints are not connected as one big blueprint (at least if someone decides to create a blueprint that connects them but that is optional) 
4) The project artifacts can also be optionally used in hyperframes (see the skill to understand more). Hyperframes in this scenario uses these artifacts to create the HTML that represents the final video, but because the artifact references are stable, if I change an artifact (by regenerating etc.) the HTML picks it up as a new artifact. So this way the user can use the blueprints to generate various artifacts and hyperframes to generate the final presentation. 
5) I am specifically targeting the blueprints to be more granular and simpler blueprints rather than all encompassing end-to-end video generation ones. So the user is expected to generate many of these for different sections of the overall video. And instead of a very rigid top level blueprint with lots of conditionals etc. these artifacts are used by AI agents and the hyperframes tool to generate a complex long running video. 
6) This is specifically targeted for long running videos anywhere from 1 minute to half hour long videos (or even longer as full feature films at some point in the future) Think of blueprints targeting various short cuts/takes of a movie. We need a way to organize these blueprints per section though as there can be many of those.
Example (How do I envision this):
Project: A documentary about the Napoleonic Wars, about 10 minutes long.
1) I would want to create a scenario: What will this cover? Which historical figures are we going to cover and talk about? Will there be sections where few experts talk about certain aspects? So in a way I want to decide on the high level scenario, characters, flow, the message, the narrative
2) Once we have a top level flow, I want to drill into sections (this can be organized hierarchically like chapters, segments etc. I.e. what is the information hierarchy of this presentation, a top level plan
3) Then for each chapter, then each segment within that etc. there can be a flow and we drill down using this hierarchy. (BTW give me some ideas from the documentary producers on how they approach this)
4) Once we nail down that planning than for each segment we can start generating the content (artifacts) needed. As a user I can have a say on how I want to present it and also get feedback from the agent on what the best way is.
5) Segment by segment all content artifacts can be prepared and iterated
6) Finally for each segment and chapter, we can use hyperframes to generate the presentation from the artifacts, narrative etc. with the style guidance the user provides.

So for each of these the user can be using blueprints to generate scenarios, character sheets, narratives etc. The user at all times would want to be able to view this as 1) Historical characters 2) Expert characters 3) The overall flow (Storyboard, chapters, segments etc.)
This is just one example. But users may want to produce anime shorts, short feature films, etc.
