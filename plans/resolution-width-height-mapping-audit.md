## Notes for Transformation Function Design
- Prefer explicit transform modes (`aspectRatio`, `preset`, `width`, `height`, `aspectRatioAndPresetObject`) whenever target schema field is non-object.
- Keep direct object pass-through only for fields whose schema accepts object (for example `video_size` / `image_size` object definitions).
- For models with strict preset enums, add explicit downgrade/auto policy with warning at transform layer (not hidden normalization).
- Resolve every `MISSING_RESOLUTION_MAPPING`, `TYPE_MISMATCH_OBJECT_TO_NONOBJECT`, `TARGET_FIELDS_NOT_IN_SCHEMA`, `UNRESOLVED_TARGET_FIELDS`, and schema lookup failure before broad migration.
