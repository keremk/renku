Asset producers are reusable execution units that generate media assets from AI models. They provide a consistent interface by mapping user-facing inputs to model-specific API parameters.

## How Producers Work

Producers abstract the complexity of working with different AI providers:

```
User Inputs (Resolution, Duration, Prompt)
    │
    │  Producer mappings apply transforms
    ↓
Provider API Fields (image_size, num_frames, prompt)
```

**Default behavior:** If a user doesn't provide a value for an input, that field is not sent to the provider. The provider uses its own defaults.

## Mapping System

Each producer defines mappings for supported providers and models. Mappings transform user-facing inputs to provider-specific API fields.

### Mapping Structure

```yaml
mappings:
  <provider>:
    <model>:
      <ProducerInput>: <mapping>
```

### Simple Mapping

Direct field rename. Supports dot notation for nested paths.

```yaml
# Flat field
Prompt: prompt
Seed: seed

# Nested field (creates { voice_setting: { voice_id: value } })
VoiceId: voice_setting.voice_id
```

## Transform Reference

Transforms convert values when producer inputs don't directly match provider API fields. There are 9 transform types.

### 1. transform - Value Lookup Table

Maps producer values to provider-specific values using a lookup table.

**Syntax:**

```yaml
ProducerInput:
  field: provider_field
  transform:
    'producer_value1': provider_value1
    'producer_value2': provider_value2
```

**Example:** Project `Resolution` to provider presets:

```yaml
# From text-to-image.yaml - bytedance/seedream/v4
Resolution:
  field: image_size
  resolution:
    mode: aspectRatio
  transform:
    '16:9': landscape_16_9
    '9:16': portrait_16_9
    '4:3': landscape_4_3
    '1:1': square_hd
```

**Example:** Map boolean to provider enum:

```yaml
EnhancePrompt:
  field: enhance_prompt_mode
  transform:
    true: standard
    false: fast
```

### 2. combine - Multiple Inputs to One Field

Merges multiple producer inputs into one provider field using composite keys.

**Syntax:**

```yaml
OutputField:
  combine:
    inputs: [Input1, Input2]
    table:
      'value1+value2': result_value
      'value1+': result_when_only_first
      '+value2': result_when_only_second
```

**Key format:** `"{Input1Value}+{Input2Value}"` - empty values allowed.

**Example:** Combine two generic inputs:

```yaml
# Example only (generic)
CodecPreset:
  combine:
    inputs: [Codec, Quality]
    table:
      'h264+fast': h264_fast
      'h264+quality': h264_quality
      'prores+quality': prores_hq
```

### 3. conditional - Include When Condition Met

Includes field only when a specific condition is satisfied.

**Syntax:**

```yaml
ProducerInput:
  conditional:
    when:
      input: OtherInput
      equals: value # OR
      notEmpty: true # OR
      empty: true
    then:
      field: provider_field
      # OR nested transform
```

**Example:** Only include a field when another input is present:

```yaml
# Generic conditional example
WatermarkText:
  conditional:
    when:
      input: BrandingText
      notEmpty: true
    then:
      field: watermark
```

**Condition types:**

- `equals: value` - Input equals specific value
- `notEmpty: true` - Input is provided (not null/undefined/empty)
- `empty: true` - Input is not provided

### 4. firstOf - Array to Single Element

Takes first element from an array input when provider expects a single value.

**Syntax:**

```yaml
ProducerInput:
  field: provider_field
  firstOf: true
```

**Example:** Single image from array:

```yaml
# From image-hybrid.yaml - qwen/qwen-image
ReferenceImages:
  field: image
  firstOf: true
```

### 4b. Schema-Driven Fan-In Projection

Renku stores fan-in values with graph metadata: groups, order, and canonical member IDs. Provider APIs do not want that graph shape. They usually want a plain array field.

Producer mappings name the target field, and the model input schema decides the payload shape.

```yaml
ReferenceImages:
  field: image_urls
```

Example:

```text
Renku fan-in value:
  groups:
    - [portraitBlob, sheetBlob]

Provider payload after schema projection:
  image_urls: [portraitBlob, sheetBlob]
```

Because the model schema says `image_urls` is an array of URI strings, Renku projects the grouped fan-in value into a plain array. The provider file resolver then uploads any local blob values in that array and replaces them with provider URLs before the SDK call.

Nested provider shapes use field paths:

```yaml
ElementFrontalImages:
  field: elements[].frontal_image_url

ElementReferenceImages:
  field: elements[].reference_image_urls
```

For a schema like Kling O3, this creates:

```yaml
elements:
  - frontal_image_url: hero-front.png
    reference_image_urls:
      - hero-side.png
      - hero-costume.png
```

If a fan-in collection is mapped to a scalar field, Renku fails validation unless the mapping explicitly uses a scalar transform such as `firstOf`. It never silently picks the first item.

If no value is connected to an optional fan-in input, the field is not sent to the provider.

Prompt labels follow final payload order. Seedance reference prompts use `@Image1`, `@Image2`, `@Video1`, `@Audio1`, and so on. Kling O3 uses both top-level image labels such as `@Image1` and element labels such as `@Element1`.

### 4c. asArray - Scalar to Array

Wraps a scalar value as a single-item array when the provider expects a list.

```yaml
SourceImage:
  field: image_urls
  asArray: true
```

### 5. invert - Boolean Inversion

Flips boolean value for providers using inverted logic.

**Syntax:**

```yaml
ProducerInput:
  field: provider_field
  invert: true
```

**Example:** EnableSafetyChecker to disable_safety_checker:

```yaml
# From image-hybrid.yaml - qwen/qwen-image
EnableSafetyChecker:
  field: disable_safety_checker
  invert: true
```

### 6. intToString - Integer to String

Converts integer to string for providers expecting string enums.

**Syntax:**

```yaml
ProducerInput:
  field: provider_field
  intToString: true
```

**Example:** Duration as string:

```yaml
# Duration 5 becomes "5"
Duration:
  field: duration
  intToString: true
```

### 7. intToSecondsString - Integer to Seconds String

Converts integer to string with "s" suffix.

**Syntax:**

```yaml
ProducerInput:
  field: provider_field
  intToSecondsString: true
```

**Example:** Duration with seconds suffix:

```yaml
# From image-to-video.yaml - veo3.1/image-to-video
# Duration 8 becomes "8s"
Duration:
  field: duration
  intToSecondsString: true
```

### 8. durationToFrames - Seconds to Frame Count

Converts duration in seconds to frame count based on fps.

**Syntax:**

```yaml
ProducerInput:
  field: provider_field
  durationToFrames:
    fps: 24
```

**Example:** Duration to num_frames at 24fps:

```yaml
# From audio-to-video.yaml - infinitalk
# Duration 5 seconds becomes 120 frames
Duration:
  field: num_frames
  durationToFrames:
    fps: 24
```

### 9. expand - Spread Object Into Payload

When a transform produces an object, spread its properties directly into the payload instead of nesting under a field name.

**Syntax:**

```yaml
OutputField:
  input: Resolution
  resolution:
    mode: aspectRatioAndPresetObject
    aspectRatioField: aspect_ratio
    presetField: resolution
  expand: true
```

With `expand: true`, the generated object is spread directly into the payload instead of nesting it under `OutputField`.

## Transform Application Order

When multiple transforms are specified, they apply in this order:

1. **conditional** - Skip if condition not met
2. **combine** - Merge multiple inputs
3. **firstOf** - Extract first from array
4. **asArray** - Wrap scalar as array
5. **invert** - Flip boolean
6. **intToString** - Convert to string
7. **intToSecondsString** - Convert to string with "s"
8. **durationToFrames** - Multiply by fps
9. **transform** - Value lookup
10. **expand** - Spread object into payload

## Creating Custom Producers

Producer YAML files follow this structure:

```yaml
meta:
  name: Producer Name
  description: What this producer does
  id: ProducerIdInPascalCase
  version: 0.1.0
  author: Your Name
  license: MIT

inputs:
  - name: InputName
    description: What this input does
    type: string | integer | number | boolean | image | video | audio | array
    itemType: image # For array types

outputs:
  - name: OutputName
    description: What this produces
    type: image | video | audio | json

mappings:
  provider-name:
    model/path:
      InputName: api_field
      # Or with transforms...
```

Place custom producers in `catalog/producers/asset/` or `catalog/producers/composition/` and reference them in blueprints using `producer: asset/your-producer` or `producer: composition/your-producer`.
